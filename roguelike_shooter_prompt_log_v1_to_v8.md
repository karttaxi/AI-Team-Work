# Roguelike Shooter 바이브코딩 프롬프트 기록

이 문서는 `roguelike_shooter.py` 초안부터 `roguelike_shooter_fixed_v8.py`까지 개선해 온 과정을 정리한 프롬프트 기록입니다.  
기말과제에서 **바이브코딩 기반으로 어떤 식으로 코드를 개선했는지** 설명하기 위한 용도로 작성했습니다.

> 핵심 방향:  
> 처음에는 간단한 pygame 슈팅 게임 초안으로 시작했고, 이후 직접 실행해 보면서 발견한 문제점과 조원/AI 피드백을 바탕으로 버그 수정, 기능 추가, 시각 피드백 개선을 반복했습니다.

---

## 0. 프로젝트 목표 설정 프롬프트

```text
Python pygame으로 간단한 로그라이크 슈팅 게임을 만들고 싶어.

조건:
1. 플레이어는 WASD로 이동한다.
2. 적은 화면 바깥에서 생성되어 플레이어를 추적한다.
3. 플레이어는 적을 공격할 수 있다.
4. 적을 처치하면 경험치 구슬이 떨어진다.
5. 경험치를 모으면 레벨업하고 업그레이드를 선택할 수 있다.
6. HP, 레벨, 웨이브, 경험치 UI가 화면에 표시된다.
7. pygame만 설치하면 실행되도록 한 파일 코드로 작성한다.
8. 초보자도 이해할 수 있도록 구조를 깔끔하게 짜고 주석을 달아준다.
```

## 2. v2 — 핵심 버그 수정 및 구조 안정화 프롬프트

```text

6. 피격 시 피해량 표시가 없음
7. 무기 최대 레벨 제한이 없음

추가 조건:
- P키로 일시정지 가능하게 해줘.
```

## 4. v3 — 시각 피드백 및 성능 개선 프롬프트

```text
게임을 플레이해 보니 다음 부분을 개선하고 싶어.
```

## 발표에서 설명할 수 있는 포인트

### 1. 바이브코딩 활용 방식

단순히 코드를 한 번에 생성한 것이 아니라, 실행 결과를 직접 확인하고 문제점을 찾아가며 프롬프트를 반복적으로 수정했습니다.

예시:
- “관통 무기가 제대로 작동하지 않는다”
- “보스 HP를 확인하기 어렵다”
- “UI나 피격 효과 등 시각 피드백이 부족하다”
- “엘리트 적이 일반 적과 차별점이 부족하다”

이런 식으로 플레이 테스트 결과와 코드 검토 결과를 바탕으로 다시 프롬프트를 작성해 개선했습니다.

### 2. 사용한 파이썬 개념

- 클래스와 객체
- 리스트와 딕셔너리
- 조건문과 반복문
- 함수 분리
- 충돌 판정
- 랜덤 생성
- pygame 화면 출력
- 이벤트 처리
- 게임 상태 관리

### 3. 최종 구현 기능

- 자동 공격
- 다양한 무기와 강화
- 경험치 및 레벨업
- 아이템 드롭
- 방어막, 흡혈, 냉기 장치
- 일반 적, 빠른 적, 탱커, 엘리트, 보스
- 엘리트 돌진 패턴
- 보스 HP바
- 일시정지 및 재시작
- 피해량 표시 및 시각 피드백

## 최종 결론

초기에는 단순한 pygame 슈팅 게임 초안이었지만, 여러 차례의 프롬프트 수정과 실행 테스트를 거치면서 로그라이크 요소가 포함된 생존형 자동 슈팅 게임으로 발전했습니다.

특히 v1부터 v8까지의 과정은 다음 흐름으로 정리할 수 있습니다.

```text
기본 구현
→ 버그 수정
→ 시각 피드백 개선
→ 무기/패시브 추가
→ 적 종류와 패턴 추가
→ 코드 구조 정리
```

현재 v8은 실행 가능한 기본 구현본이자, 조원들과 함께 도트 스프라이트, UI 디자인, 발표 자료, 실행 테스트를 이어가기 위한 기준안입니다.
